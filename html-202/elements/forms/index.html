Form Elements Done Right

Forms are the primary way users send data to your server. Getting the HTML right makes them accessible, keyboard-navigable, and resilient — before a single line of JavaScript runs.

Labels Are Not Optional

Every input deserves a <label>, connected either by nesting or via the for attribute. Screen readers use this association to announce the field's purpose, and clicking the label focuses the input — a larger touch target for free.

Native Validation First

Before reaching for a validation library, check what the browser gives you for free:

  • required — the field must be filled before submission
  • type="email" — the browser checks format on mobile keyboards
  • minlength / maxlength — character count constraints
  • pattern — a regex for custom validation rules
