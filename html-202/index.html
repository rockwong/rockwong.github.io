HTML-202: Semantic Markup

Semantic HTML is more than a best practice — it's the bedrock of an accessible, SEO-friendly, and maintainable web. Choosing the right element for your content communicates meaning to browsers, assistive technologies, and search engines alike.

<header><nav><main> <article><section><aside> <footer><figure><time>

Why It Matters

A screen reader navigating a page built with <div> elements alone has no landmarks to latch onto. The same page structured with <nav>, <main>, and <article> becomes skimmable — users can jump directly to the content they need. Search engines reward this clarity too, using semantic cues to understand page hierarchy and relevance.

A Practical Heuristic

Before reaching for a <div>, ask: does HTML already have an element for this purpose? If your content can stand alone as a syndicated piece, it's an <article>. If it groups thematically related content under a heading, it's a <section>. If it's tangentially related, it's an <aside>. The right element is usually there — you just need to look for it.
