Responsive Design Notes

Responsive design is not about making things "look good on mobile" — it's about building interfaces that adapt gracefully across the full spectrum of devices. Here are a few patterns I rely on.

Start with the viewport

Every responsive page needs a proper viewport meta tag. Without it, mobile browsers will render the page at a typical desktop width and then scale it down — making text unreadable and touch targets too small. 

<meta name="viewport" content="width=device-width, initial-scale=1">

Fluid over fixed

Instead of setting width: 960px, use max-width with a relative unit. Combine it with percentage-based columns so the layout breathes. A container that is max-width: 1190px with width: 100% gives you a ceiling without cutting off content on narrower screens.

Breakpoints follow content

Don't pick breakpoints based on device widths. Resize the browser until the layout actually breaks — then add a media query at that value. This keeps your CSS responsive to your content, not to a snapshot of the device market.

Minimal, meaningful typography

Font sizes should scale comfortably for reading on any screen. I favour system font stacks — they load instantly, feel native on each platform, and save bandwidth. A base size of 16px with generous line-height gets you most of the way there.
