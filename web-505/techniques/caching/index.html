Caching Strategies for the Web

Caching is the single most effective performance optimization available to web developers. A well-configured cache keeps repeat visits fast and reduces server load with almost no ongoing cost.

Cache-Control Headers

The Cache-Control response header is your primary tool. It tells browsers and intermediate proxies how long they can serve a resource without checking back with the origin server.

DirectiveMeaning
max-age=31536000Cache for one year (ideal for hashed assets)
no-cacheAlways validate before using cached copy
no-storeNever cache — use for sensitive data
publicCacheable by browsers and CDNs alike
immutableThe resource will never change (use with hashed filenames)

Hashed Filenames

Bundle your assets with content hashes in the filename — main.a8f3b2c.js instead of main.js. When the content changes, the filename changes, which makes the old cached copy irrelevant. Combined with max-age and immutable, this is as close to perfect caching as the web gets.

Service Worker Caching

For offline support and fine-grained control, a service worker can intercept network requests and serve from a local cache. This is the engine behind progressive web apps — and it works even when the user has no connection at all.
