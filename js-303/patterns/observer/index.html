Observer Pattern in JavaScript

The observer pattern is one of the most useful design patterns in a front-end developer's toolkit. It decouples the thing that changes from the things that react to the change — the foundation of event systems, reactive frameworks, and pub-sub architectures.

The Core Idea

A subject maintains a list of dependents (observers) and notifies them automatically when its state changes. Observers can subscribe and unsubscribe at runtime, so the wiring is dynamic rather than hard-coded.

A Minimal Implementation

class Subject {
  constructor() {
    this.observers = [];
  }
  subscribe(fn) {
    this.observers.push(fn);
    return () => {
      this.observers = this.observers.filter(o => o !== fn);
    };
  }
  notify(data) {
    this.observers.forEach(fn => fn(data));
  }
}

const store = new Subject();
const unsub = store.subscribe(data => console.log('Got:', data));
store.notify({ type: 'UPDATE', value: 42 });
unsub();

When to Use It

Reach for the observer pattern when one piece of state needs to inform multiple, loosely-coupled consumers — a search input that updates a result list, a filter panel, and a URL query string all at once. It's also the mental model behind addEventListener, MutationObserver, and Redux-style state management.
